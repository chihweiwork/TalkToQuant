## 投影片 2：問題場景

### 標題
傳統量化回測的三大痛點

### 內容

#### 痛點 1：技術門檻高 ❌
- 需要程式能力（Python/R）
- 需要了解 API 使用方式
- 學習曲線陡峭

#### 痛點 2：參數調整麻煩 ❌
- 每次調參數都要改程式碼
- 重複執行浪費時間
- 容易出錯（預設值不符預期）

#### 痛點 3：結果難以比較 ❌
- 缺乏標準化報告格式
- 數據分散在終端輸出
- 難以快速判斷策略好壞

### 視覺化建議
- 顯示傳統回測的程式碼截圖（複雜度）
- 用 ❌ 符號強調痛點

## 投影片 6：技術架構（給技術受眾）

### 標題
模組化設計 · 可擴充架構

### 四層架構

#### 1. 資料層 (`backtest/data.py`)
- FinMind API 整合
- 股票價格 / 大盤指數
- 股票代號查詢（中文 → ID）

#### 2. 指標層 (`backtest/indicators.py`)
- 移動平均線（MA）
- 相對強弱指標（RSI）
- MACD
- **可擴充**：自訂指標

#### 3. 引擎層 (`backtest/engine.py`, `portfolio.py`)
- 單一股票回測
- 投資組合回測
- Signal-driven 設計

#### 4. 報表層 (`backtest/report.py`)
- 終端摘要
- Plotly 互動圖表
- HTML 報告生成

### 技術棧
- **語言**：Python 3.12+
- **資料處理**：Pandas, NumPy
- **視覺化**：Plotly
- **資料來源**：FinMind API

## 投影片 7：進階功能 - 投資組合回測

### 標題
不只單一股票，支援投資組合分析

### 範例設定
```python
# 台積電 50% + 鴻海 25% + 聯發科 25%
stocks = {
    "2330": "台積電",
    "2317": "鴻海", 
    "2454": "聯發科"
}

weights = {
    "2330": 0.5,
    "2317": 0.25,
    "2454": 0.25
}
```

### 回測結果
```
投資組合績效：
  總報酬：+25.83%
  Sharpe Ratio：1.42
  
各股損益：
  台積電：+207,640 元
  鴻海：-240,867 元（虧損）
  聯發科：+116,485 元
```

### 關鍵洞察
- ✅ 分散風險（鴻海虧損但組合仍獲利）
- ✅ 各股可用不同策略
- ✅ 自訂權重分配
